# Contributing to AlgoArena

AlgoArena에 참여해주셔서 감사합니다! 아래 규칙을 읽고 따라주세요.

---

| 닉네임 | 이름 |
|--------|------|
| Roel | 세홍 |
| Hyerim | 혜림 |
| Yukyung | 유경 |
| Hyeonjun | 현준 |

---

## 📁 폴더 구조

```
AlgoArena/
├── Roel/
│   ├── BOJ/
│   │   └── 1234.py
│   └── Programmers/
│       └── 문제이름.py
├── Hyerim/
├── Yukyung/
└── Hyeonjun/
```

- 본인 닉네임 폴더 아래에 플랫폼별로 분리해서 올려주세요.
- 파일명은 **백준: 문제번호** / **프로그래머스: 문제이름** 으로 작성해 주세요.

---

## 🌿 브랜치 규칙

```
닉네임/플랫폼-문제번호
```

**예시:**
```
Roel/BOJ-1234
Hyerim/Programmers-두_큰수
```

- `main` 브랜치에 직접 push하지 않습니다.
- 항상 본인 브랜치를 만들어서 PR로 올려주세요.

---

## 💬 커밋 메시지 규칙

```
[플랫폼 문제번호] 문제 제목
```

**예시:**
```
[BOJ 1234] 마법사 상어와 파이어볼
[Programmers] 두 큰수의 합
```

---

## 🔃 PR 규칙

1. PR 제목은 커밋 메시지 규칙과 동일하게 작성해 주세요.
2. PR 템플릿에 맞게 내용을 채워주세요.
3. **최소 2명의 리뷰 승인** 후 머지합니다.
4. 본인 PR은 본인이 머지합니다. (리뷰 승인 후)

---

## 🗣️ 코드 리뷰 가이드

- 리뷰는 **비판이 아닌 성장**을 위한 것입니다.
- 다른 풀이 방식이 있다면 적극적으로 공유해 주세요.
- 리뷰 코멘트는 아래 prefix를 활용해주세요.

| prefix | 의미 |
|--------|------|
| `[질문]` | 코드 이해가 안 될 때 |
| `[제안]` | 더 나은 방법이 있을 때 |
| `[칭찬]` | 좋은 풀이 공유 |
| `[오류]` | 버그나 엣지케이스 발견 |

---

## 🐍 언어 규칙

- **Python 권장** (Python 3 기준)
- 다른 언어(Java, C++ 등)도 가능하며, 파일 확장자로 구분합니다.

---

## 📅 참여 방법 요약

```
1. 이슈에서 이번 주 문제 확인
2. 브랜치 생성 (닉네임/플랫폼-문제번호)
3. 본인 폴더에 풀이 파일 추가
4. PR 생성 후 이슈 연결
5. 2명 리뷰 완료 후 머지
```
